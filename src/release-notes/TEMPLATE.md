# Release Notes Template

```md
# Release [version-or-range]

## Highlights
- [Most important customer-visible improvement]
- [Second major item]

## Features
- [Feature outcome]

## Fixes
- [Bug fix outcome]

## Performance
- [Perf improvement and expected impact]

## Developer Experience
- [Tooling, docs, CI/CD, test improvements]

## Breaking Changes
- [Change]
- Migration: [what users must do]

## Known Issues
- [Only include if confirmed]
```

If a section is empty, omit it except `Highlights`.
