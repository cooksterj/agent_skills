# Pull Request Review Template

Use this structure for review output.

```md
## Verdict
- [approve | request changes | comment]

## High
- [File/path]: [Issue], [impact], [recommended fix]

## Medium
- [File/path]: [Issue], [impact], [recommended fix]

## Low
- [File/path]: [Issue], [impact], [recommended fix]

## Test coverage notes
- [Missing tests or confidence statement]

## Residual risks
- [Any deployment/operational risk left after merge]
```

If a section has no findings, write `- None`.
