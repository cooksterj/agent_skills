# Pull Request Description Template

Use this format when drafting pull requests.

## Title

```
<type>: <concise summary>
```

## Body

```md
## Summary
- [Primary change]
- [Secondary change]
- [Any notable cleanup or docs update]

## Why
- [Problem being solved]
- [User or system impact]

## Testing
- [x] [Test command and scope]
- [ ] [Any unexecuted test and reason]

## Risks and rollout
- [Risk and mitigation, if any]
- [Rollback note, if applicable]

## Checklist
- [x] Self-review complete
- [x] Docs updated (if needed)
- [x] No secrets added
```

## Notes

- Omit empty sections if truly not applicable, except `Summary` and `Testing`.
- Keep each bullet one sentence where possible.
